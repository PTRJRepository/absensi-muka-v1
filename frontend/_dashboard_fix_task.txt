# Fix DashboardPage — Quality API Mismatch

## PROBLEM
Dashboard page crashes: `Cannot read properties of undefined (reading 'healthy_count')`

The backend `/api/quality/summary` returns:
```json
{
  "success": true,
  "data": {
    "period_days": 7,
    "total_scans": 50152,
    "mapped_employees": 1409,
    "unmapped_codes": 716,
    "failed_batches": 104,
    "completed_batches": 0,
    "status": "WARNING"
  }
}
```

But DashboardPage expects `QualityReport` type:
```typescript
interface QualityReport {
  score: number;
  overall_status: 'healthy' | 'warning' | 'critical';
  summary: { healthy_count: number; warning_count: number; critical_count: number; };
  metrics: Array<{ name: string; status: 'healthy' | 'warning' | 'critical'; value: number }>;
}
```
